### 一句话总结

提出一种双分支 CNN 架构，分别处理 FMCW 雷达的 1D 时间序列信号和 2D 图像表示（Range-Doppler Map 等），通过特征融合提升人体检测精度。

### 研究痛点与动机

- **传统做法：** 雷达人体检测通常选择单一数据表示——要么直接处理 1D 时间序列（如原始 IF 信号、距离剖面），要么将数据转换为 2D 图像（如 Range-Doppler Map、微多普勒频谱图）再用 CNN 处理。
- **痛点：** 单一表示会丢失互补信息。1D 时间序列保留了精确的时域波形和相位信息，但缺乏空间结构；2D 图像具有丰富的空间/频率模式，但经过变换后丢失了部分原始信号细节。
- **切入点：** 同时利用两种表示的互补优势，通过多模态 CNN 融合提升检测鲁棒性。

### 核心创新与方法论

- **双分支架构 (Dual-Branch Architecture)：**
  - **1D 分支：** 处理雷达原始时间序列（如距离剖面、相位序列），使用 1D CNN 提取时域特征。
  - **2D 分支：** 处理雷达图像表示（如 Range-Doppler Map、微多普勒频谱图），使用 2D CNN 提取空间/频率特征。
- **特征融合 (Feature Fusion)：** 在中间层或末端将两个分支的特征进行融合（拼接/加权/注意力融合），输出检测结果。
- **与 [[RadHAR]] 的对比：** RadHAR 只用 3D 体素化的空间特征。本文同时利用 1D 和 2D 两种表示，信息更完整。

### 输入/输出 (I/O)

- **Input：**
  - 分支 1：1D 雷达时间序列信号
  - 分支 2：2D 雷达图像（Range-Doppler Map / 微多普勒频谱图）
- **Backbone：** 双分支 CNN + 特征融合层
- **Output：** 人体检测结果（存在/不存在，或包含定位信息）

### 批判性思考

- **会议级别：** ESCI 是新兴会议，学术影响力相对有限，但技术思路有参考价值。
- **多模态融合的启示：** 本文的核心思路——同时利用 1D 和 2D 两种雷达表示——可以推广到更复杂的任务（如动作识别、姿态估计）。例如，在 [[RadMamba]] 的微多普勒频谱图基础上，额外引入 1D 相位序列作为辅助输入。
- **局限：** 双分支架构增加了计算开销，可能不适合边缘部署。融合策略的选择（早期融合 vs 晚期融合）对性能影响较大。

### 对我的课题的帮助

- **多模态输入思路：** 可以借鉴双分支设计，在毕设中同时利用雷达的点云（空间）和微多普勒频谱（时间-频率）两种表示。
- **融合策略参考：** 其特征融合方法可以与 [[Star Graph + DDGNN]] 的图特征 + LSTM 时序特征融合方式进行对比。
