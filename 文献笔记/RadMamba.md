#毫米波 #行为识别 
### 论文标题：
RadMamba: Efficient Human Activity Recognition Through a Radar-Based Micro-Doppler-Oriented Mamba State-Space Mode
### 标题翻译
基于雷达微多普勒特征的 Mamba 状态空间模型实现高效人体活动识别

_IEEE Transactions on Radar Systems_ (IEEE T-RS) 2025.12

###  **研究痛点 (Motivation):** 
- **传统做法：** 处理雷达信号（微多普勒频谱或点云）通常用 CNN、LSTM 或 Transformer。（例如经典的基线架构 [[RadHAR]]）或 Transformer。
- **痛点在哪？** Transformer 虽然牛，但它的自注意力机制计算量是$O(N^2)$。雷达只要连续监测几秒钟，序列$N$就会非常大，导致模型根本无法塞进智能家居的微型雷达芯片里。
### **核心创新 (Method):** 
- 作者将大模型领域最新的 Mamba (状态空间模型 SSM) 引入了雷达领域。
- 他们针对雷达微多普勒（Micro-Doppler, MD）信号的物理特性，定制了专门的 Mamba 块，实现了$O(N)$的线性计算复杂度。也就是：既有 Transformer 的全局视野，又像 RNN 一样轻量快速。
### 输入/输出 (I/O Shape):
 - **Input (输入):** 雷达原始数据经过 FFT 处理后的**微多普勒时频图 (Spectrograms)**。在计算机眼里，这本质上是一个时间步为 $T$ ，特征维度为 $D$ 的时序序列，形状大概是 [ Batch_size, T, D ] 或者二维图 [ Batch_size, Channels, Freq, Time ]。
- **Backbone (骨干):** 一系列的 RadMamba Block。
- **Output (输出):** 动作分类的概率分布（比如：走、跑、跌倒的概率）。

### 阅读过程
mamba要求输入为一维数据，而雷达数据通常是输入一个微多普勒时频图。
舍弃了传统 ViT/Mamba 破坏物理结构的 Linear 线性拉平投影（会丢失一维速度特征），改用 Conv1d (一维卷积) 进行 Token 投影。这不仅保住了多普勒频率的局部连续性，还大大降低了参数量。
在连续 FMCW 雷达数据集上，**仅用 6.7k 个参数**（参数量是同类 SSM 的 1/400 甚至 1/10），准确率反超其他大模型达到 SOTA (最高99.8%)。
传统的Vision Mamba (ViM)运用到毫米波雷达会有==三个问题==：
1) 问题1：微多普勒谱具有显著的背景稀疏性，因此大多数时频单元几乎不包含与任务相关的信息。 
2) 问题2：矩形区域分割会破坏微多普勒模式的连续时空结构，从而降低表征质量。 
3) 问题3：标准的线性投影层难以捕捉多普勒特有的时频模式。

#### 1. 机制一：通道融合与降采样 (Channel Fusion & Downsampling)
现在的雷达通常有多个接收天线（多通道）。如果把所有通道的数据都输入进去，计算量太大。所以作者第一步先做了一个融合，把多通道的冗余信息合并，并且把图中“没有动作的低价值空白区域”缩小，这一步极大地减少了后续计算的负担。
#### 2. 机制二：多普勒对齐切片 (Doppler-Aligned Segmentation)
![[Pasted image 20260330214313.png]]
- 这是解决“怎么把图片切成小块（Token）”的问题。传统的 ViT 横着切一刀竖着切一刀，变成正方形的 Patch（比如 16x16） 如（a）。但雷达图的纵坐标是“多普勒速度”，如果横向切断了，速度和时间的连续性就被破坏了。
- **作者的做法：** 他们切片的时候，**顺着多普勒的频率方向切（Doppler-aligned）**，保证速度维度的物理意义是完整的。

切片具体的理解等
	一个一个小块，会重复出现 $$[t_0,t_1...,t_j,t_o...t_j]$$这样的时间序列，这种不连续性，从而使得 SSM 难以提取有意义的时间动态特征
	同理，多普勒频率也是如此，会出现 j 到 0 的跳跃
#### 3. 机制三：一维卷积 Token 投影 (Convolutional Token Projections)
这是 RadMamba 核心模块 **CP-Mamba Block** 的关键创新。在传统的 Mamba 块中，数据特征的映射通常依赖于**线性投影层（Linear）**，但线性层只能孤立地处理单一时间节点，容易割裂雷达动作的物理连续性。为此，作者**将 Mamba 块内部的输入与输出投影层全部替换为了核大小为 3 的一维卷积（Conv1d）**。这种“滑动扫描”机制使得模型在处理当前切片时，能够自然地融合前后相邻的时间信息，不仅有效捕捉了局部上下文，还极大增强了同一动作切片之间的“时序连贯性”（在数学上表现为交叉相关性的显著提升）。

关于卷积增加相关性
	通过实验，一个人一直反复动作，查看batch间的相关性，更换后得到了很大的提升。

#### 4. Mamba 状态空间骨干网络 (The SSM state-space model（状态空间模型） Backbone)
==**CP-Mamba Block** 内部的核心组件==
- RadMamba 的核心在于利用了**选择性状态空间模型 (Selective SSM)**。它通过一个可学习的步长 $\Delta$，将连续的状态转移矩阵离散化 。
- 与传统 RNN 不同的是，它的参数（如 $B$、$C$ 矩阵和步长 $\Delta$）是**依赖于输入的 (Input-dependent)**，通过线性投影从输入动态生成 。这使得模型能够根据微多普勒序列的动态特性，自适应地“选择”记住关键的动作特征，并忽略无效的背景噪声。

### 实验结果
![[Pasted image 20260409214826.png]]
- **参数**：在 DIAT 数据集上，RadMamba 达到了 99.8% 的准确率，仅需 21.7k 个参数，这只有近期基于 SSM 模型的 1/400 。
- **算力**：在动作更多的 CI4R 数据集上，取得了 91.2% 的准确率，但每样本推理的浮点运算数 (#FLOP/Inf.) 仅为 8.8M，远低于之前 Transformer 模型的 169M 至 2410M 。
- **连续动作：在最复杂的连续序列 UoG20 上，RadMamba 以极小的 6.7k 参数量和 1.0M #FLOP/Inf. 实现了 89.3% 的准确率，击败了参数量大得多的基线模型 。
### 数据集
数据集 (Datasets) 论文在三种复杂程度递进的雷达数据集上进行了验证：

1. **[[DIAT]]**：10 GHz 连续波 (CW) 雷达，包含 6 类**非连续**的可疑人类活动 。
2. **[[CI4R]]**：77 GHz FMCW 雷达，包含 11 类**非连续**的日常动作和步态 。
3. **[[UoG20]]**：5.8 GHz FMCW 雷达，包含 6 类**连续**动作序列（极具挑战性，因为动作之间没有明确边界）。

复现出现问题
- 原代码数据源为 2D 独热编码（One-Hot），而 PyTorch 默认的 `CrossEntropyLoss` 拒绝接收多目标（multi-target）张量。
- **解决方案**：在计算 Loss 前，引入 `.argmax(dim=1)` 算子，将 2D 的 One-Hot 标签强制转换为 1D 的类别索引（Class Indices），实现张量降维，对齐 PyTorch 底层 API。
- **代码修改点**：
	`modules/train_funcs.py` 中的 `loss = criterion(out, output.argmax(dim=1))`


- **情况 A（忘了推代码）**：作者在自己电脑上跑的时候，也遇到了这个报错。于是他随手在自己电脑的 `Dataset` 类或者 `train_funcs.py` 里加上了 `.argmax(dim=1)` 修复了这个问题，然后论文顺利跑出结果。但在最后把代码传到 GitHub 开源时，他**漏传了这几行修改**，直接把旧版本推上去了。
- **情况 B（换了损失函数忘了改回来）**：作者一开始可能用的是支持多目标的 `BCEWithLogitsLoss`（二元交叉熵），所以配合独热编码没问题。后来发论文前，为了效果更好，改成了传统的 `CrossEntropyLoss`，但他忘记同步修改数据标签的格式了。

> [!success] 成功的验证点

> - **强正则化**：原作者可能在 71.4k 的配置中使用了更高的 Weight Decay 或引入了额外的 Dropout 层来抑制过拟合。
> - **已经超越论文中的部分强力 Baseline（如 Bi-LSTM 的 79.4% 和 CNN-Bi-GRU 的 84.9%）。
